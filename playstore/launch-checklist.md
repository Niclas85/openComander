# Launch Checklist

## Store Trust

- App is clearly described as free and open source.
- Listing says no ads, no tracking and no account only if this remains true.
- Privacy policy URL is public and matches the actual app behavior.
- Contact email is set to `info@opengames.vip`.
- Provider postal address is final.
- `legal/privacy-policy-public-template-de.md` is filled and published.
- Screenshots do not show private file names or copyrighted content.

## Open Source Trust

- Public repository is available: https://github.com/Niclas85/openComander
- License file is present.
- README explains build steps.
- Issues are enabled.
- Contribution expectations are clear.
- Release tags are used for published builds.

## Google Play Readiness

- Data Safety answers match the app: no analytics, no ads, no account, no provider upload.
- Data Safety answers are prepared in `playstore/data-safety-de.md` and `playstore/play-console-answers-de.md`.
- All Files Access declaration is prepared in `playstore/all-files-access-de.md` and `playstore/play-console-answers-de.md`.
- Store listing avoids third-party trademarks in promotional claims.
- Screenshots show real product UI.
- Feature graphic and icon match the app name.

## Community Launch

- GitHub release with APK or Play link.
- Short demo clip for dual-pane copy/move.
- Short demo clip for undo history.
- Short demo clip for ZIP browsing.
- Post a transparent build log, not a hype post.
- Invite feedback on specific features.
